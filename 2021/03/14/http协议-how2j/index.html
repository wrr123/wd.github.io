协议概念

HTTP 超文本传输协议 Hyper Text Transfer Protocol

当我们在浏览器的地址栏输入一个地址的时候，就能够访问服务器的某个页面。

这个过程本身就是两个应用程序之间的交互，一个应用程序是浏览器，另一个应用程序是服务器。

协议是什么？

  • 它就是不同的应用程序之间按照事先做好的约定进行的通信。这样能互相读懂对象的意思。

浏览器和WEB服务器之间，使用的就是一种叫HTTP的协议。

这是BS(browser server)架构模型的基础。

HTTP协议由请求和响应两种类型组成。

调试工具

请求协议

请求消息由请求行、请求头部、空行和请求数据4部分组成。

请求头部信息提供了如下信息:
Host: 主机名
User-Agent: 浏览器基本资料
Accept: 浏览器能够识别的响应类型
Accept-Language: 浏览器默认语言
Accept-Encoding: 浏览器能够识别的压缩方式
Referer: 来路页面， /addHero 这个路径是通过addHero.html这个页面跳转过来的。
Connecton：是否保持连接

响应协议

响应消息由状态行、消息报头和响应正文组成。

消息报头中提供如下信息：
Content-Length: 表示长度
Content-Type: 内容格式
Date: 日期
Server: 服务器类型

响应代码

http协议中的响应代码从1xx~5xx,一共有41种，不过很多状态码很少会接触到。

一些常见的状态码：

  • 200 响应成功了
  • 301 客户端跳转，永久性的跳转
  • 302 客户端跳转，临时性的跳转
  • 304 资源未被修改
  • 404 访问的页面不存在
  • 500 服务端的错误
